# 每日遇见3位智者 — 功能规划

## 目标

在独立博客页面中，每天展示 3 位智者，每位来自不同的十大问题领域（Q01–Q10），并给每位智者分配唯一编码。

---

## 一、智者唯一编码方案

### 格式：`WP-XXXX`

- **WP** = Wise Person
- **XXXX** = 4 位数字，从 0001 开始按顺序分配

### 排序规则

按 十大问题 → 姓名拼音排序，同问题内按 name 字典序：

| 编号范围 | 问题 | 数量 |
|---------|------|------|
| WP-0001 ~ WP-0068 | Q01 知识的知识 | 68 |
| WP-0069 ~ WP-0124 | Q02 如何理解世界 | 56 |
| WP-0125 ~ WP-0176 | Q03 如何理解历史 | 52 |
| WP-0177 ~ WP-0242 | Q04 如何理解时代 | 66 |
| WP-0243 ~ WP-0303 | Q05 如何理解社会 | 61 |
| WP-0304 ~ WP-0345 | Q06 如何理解组织 | 42 |
| WP-0346 ~ WP-0391 | Q07 如何理解家庭 | 46 |
| WP-0392 ~ WP-0508 | Q08 如何理解人性 | 117 |
| WP-0509 ~ WP-0547 | Q09 如何理解身体 | 39 |
| WP-0548 ~ WP-0593 | Q10 如何理解信仰 | 46 |

> 跨领域智者（属于多个问题）在首次出现时分配编码，再次出现在其他问题时使用同一编码，不重复分配。

### 数据结构

新增 `src/data/wise-person-codes.json`：

```json
{
  "WP-0001": {
    "slug": "plato",
    "name": "柏拉图",
    "primaryQuestion": 1,
    "allQuestions": [1]
  },
  ...
}
```

## 二、每日3位智者的轮播算法

### 维度轮换规则

每天 3 位智者，从不同维度选取：

| 维次 | 包含问题 |
|------|---------|
| **元** | Q01 知识的知识 |
| **天** | Q02 如何理解世界、Q03 如何理解历史、Q04 如何理解时代 |
| **地** | Q05 如何理解社会、Q06 如何理解组织、Q07 如何理解家庭 |
| **人** | Q08 如何理解人性、Q09 如何理解身体、Q10 如何理解信仰 |

### 轮播策略

采用 **Round-Robin 轮播** + **维度内随机**：

1. 每天从 4 个维度中选取 3 个（三个维度轮换，第四个维度次日优先）
2. 每个维度内从未展示过的智者和已展示过的智者分别维护，优先展示未展示的
3. 全部展示完后循环

### 进度持久化

每个用户的进度以 cookie 或 localStorage 存储：
- `lastDate`: 上次访问日期
- `shownIds`: 已展示过的智者编码列表
- `dailyHistory`: 历史每日记录

## 三、页面设计

### 路由

`/daily` — 每日遇见页面

### 布局

```
┌─────────────────────────────────────────────┐
│  每日遇见                                    │
│  2026年5月20日 · 第 X 天                     │
│                                             │
│  ┌─ WP-0042 ──────────────────────────┐    │
│  │  Q01 知识的知识 · 元               │    │
│  │  卡尔·波普尔 / Karl Popper        │    │

│                                             │
│  ┌─ WP-0156 ──────────────────────────┐    │
```

### 每个智者卡片包括

- WP 编码（左上角小标签）
- 问题编号 + 标题 + 维度
- 姓名（中英文）
- 一句话摘要（从 life-stories 中提取首句）
- "查看详情" 链接 → `/wise-persons/[slug]`
