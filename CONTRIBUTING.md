# 参与贡献

欢迎贡献智者内容！你可以为已有的智者补充资料，也可以添加新的智者。

## 贡献方式

### 1. 提交新的智者

在 `智者资料库/` 下创建以智者英文 slug 命名的目录，包含 3 个 Markdown 文件：

```
智者资料库/{slug}/
├── 01-智者介绍.md    # 生平与思想（核心）
├── 02-基本信息.md    # 年表、著作、概念清单
└── 03-认知方式.md    # 认知方式分析
```

参考模板：`智者资料库/00-模板文件/`

> 模板仅供参考，如果你有独特的介绍方式（访谈体、故事体、图解式等），欢迎贡献创意——形式服务于内容，让智者"活"起来更重要。

### 2. 补充已有智者

直接修改对应目录下的 MD 文件即可。

### 3. 更新 metadata.json

新增或修改智者后，请同步更新 `智者资料库/metadata.json` 中的 `wise_people` 数组：

```json
{
  "slug": "zhang-san",
  "name": "张三",
  "english_name": "Zhang San",
  "years": "1900—1980",
  "questions": ["Q01"],
  "topics": ["关键词1", "关键词2"],
  "status": "complete",
  "has_introduction": true,
  "has_sources": true,
  "has_cognitive_style": true
}
```

`questions` 从以下编号中选择：Q01(知识的知识)、Q02(如何理解世界)、Q03(如何理解历史)、Q04(如何理解时代)、Q05(如何理解社会)、Q06(如何理解组织)、Q07(如何理解家庭)、Q08(如何理解人性)、Q09(如何理解身体)、Q10(如何理解信仰)。

### 4. 提交 PR

- fork 本仓库
- 在 `main` 分支上创建一个 feature 分支
- 提交你的智者文件
- 发起 Pull Request

## 文件格式说明

### 01-智者介绍.md

Markdown 格式，包含：
- 标题：`# 姓名` + `## Name｜生卒年`
- 一句引语（blockquote）
- `## 他是谁`——生平故事
- `## 他的思想`——核心思想，每条用 `###` 标题
- 结尾可选：主要著作表格、延伸阅读、与其他智者对比

### 02-基本信息.md

结构化信息清单，包含身份标签、生平年表、核心概念、主要著作表格、师承与影响等。

### 03-认知方式.md

认知方式分析，包含：
- 标题：`# 姓名的认知方式` + `## Name｜一句话概括`
- 一句引语
- 三种认知方式的标签：`**标签1 × 标签2 × 标签3**`
- 每种认知方式独立成节，用 `## 第一种认知方式：xxx` 格式
- 结尾：三种认知方式的交汇 + 延伸阅读

## 注意事项

- 所有内容使用中文
- 文件编码：UTF-8
- 段落之间不要使用 `---` 横线分隔符
- 引用来源时注明出处
- AI 生成内容需标注
- **所有贡献内容请在文件末尾附上贡献者署名和日期**，格式：`*贡献者：XXX｜YYYY年MM月DD日*`
