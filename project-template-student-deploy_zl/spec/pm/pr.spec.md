# 智者网（WisePeople）— 产品需求文档（可供性驱动）

## 1. 产品环境

**名称：** 智者网（WisePeople）
**标语：** 为成年人打造的通识教育地图
**版本：** 1.0.0

**环境描述：**
智者网是一个面向终身学习者的通识教育平台。平台围绕"十大问题导览、智者库、代表作索引、最小限度书单"四大板块，为用户提供系统性构建跨学科知识框架的行动空间。用户在此环境中探索智者的思想世界，发现高质量阅读路径，并逐步从知识消费者转变为智慧创作者。

**主要智能体：**

- **普通用户（浏览者/注册读者/内容贡献者）**：浏览智者库、阅读书单推荐、撰写书评笔记、收藏内容
- **内容管理员（编辑/审核员/超级管理员）**：审核智者和著作信息、管理用户生成内容、更新推荐书单
- **AI 推荐助手**：分析用户阅读偏好、生成个性化推荐、回答知识性问题

**核心可供性：**

1. 探索智者世界 — 浏览、筛选、发现 420 位跨学科智者
2. 发现阅读路径 — 通过问题导览或书单找到阅读入口
3. 深度参与创作 — 记录笔记、撰写书评、构建个人阅读体系

## 2. 可供性目录（按 MAS 组织）

### MAS-1：探索智者世界

> **故事主题：** 用户首次进入智者网，能够感知到丰富的智者和著作信息，并自由探索这个知识环境。

**核心可供性序列：**

1. **浏览智者库** — 按时代/学科/地区浏览智者（感知：分类导航栏、智者卡片）
2. **查看智者详情** — 了解智者的生平、核心思想与贡献（感知：点击智者卡片进入详情页）
3. **浏览代表作** — 查看每位智者的推荐著作及简介（感知：著作列表，附推荐语）
4. **按问题导览** — 通过"十大问题"进入相关智者和著作（感知：问题卡片网格）

**意义闭合：**
用户从茫然不知从何入手，到建立起对知识版图的宏观感知——"原来世界可以被这样理解"。

**内在动机：**
- **兴趣：** 跨学科智者的故事天然引人入胜
- **精通：** 随着探索深入，用户逐渐理解知识谱系的内在脉络
- **自主：** 完全自由的探索路径，无强制顺序

#### 可供性：A1-浏览智者库

**级别：** 主要
**启用的操作：** 按分类浏览智者列表

**环境属性：**

| 属性 | 值 | 可被感知者 |
|----------|-------|----------------|
| 分类导航 | 学科/时代/地区 Tab 切换 | 人类 |
| 智者卡片网格 | 姓名+肖像+核心标签 | 人类 |
| 语义导航 | `<nav>` + `aria-label` 分类标识 | AI 智能体 |

**智能体要求：**

| 智能体类型 | 所需能力 |
|------------|----------------------|
| 人类 | 视觉识别、点击/触摸 |
| AI 智能体 | DOM 导航、链接访问 |

**感知-行动耦合：**

- **人类 — 看到：** 水平分类 Tab 栏，下方为智者卡片网格
- **人类 — 执行：** 点击 Tab 切换分类，滚动浏览卡片
- **人类 — 反馈：** Tab 高亮切换，卡片列表平滑过渡

- **AI — 检测：** `<nav role="tablist">` 包含分类选项
- **AI — 执行：** 通过 URL 参数 `?category=philosophy` 访问分类视图
- **AI — 验证：** 页面标题和卡片内容反映对应分类

**约束条件：**
- 智者信息必须标注来源（来自 real.md 版权合规约束）
- 内容须经人工审核确认（来自 real.md 内容审核约束）

**级别：** 主要
**启用的操作：** 查看智者详细信息及代表作

**环境属性：**

| 属性 | 值 | 可被感知者 |
|----------|-------|----------------|
| 详情页面 | 生平介绍+核心思想+代表作列表 | 人类 |
| 结构化数据 | JSON-LD 标记 | AI 智能体 |

**感知-行动耦合：**

- **AI — 执行：** 直接访问 `/wise-persons/{slug}` 端点
- **AI — 验证：** 返回包含智者姓名、简介、著作列表的完整数据

### MAS-2：发现阅读路径

> **故事主题：** 用户有了初步探索后，需要更精准的工具来找到适合自己的阅读入口。

**核心可供性序列：**

1. **搜索智者和著作** — 通过关键词搜索（感知：搜索框固定在页面顶部）
2. **浏览推荐书单** — 查看"最小限度书单"和主题书单（感知：书单卡片入口）
3. **收藏内容** — 收藏智者和书单到个人空间（感知：收藏按钮）
4. **AI 智能推荐** — 描述阅读偏好后获得个性化推荐（感知：推荐入口）

**意义闭合：**
用户从漫无目的的浏览，到找到"下一本要读的书"——探索开始聚焦。

**内在动机：**
- **兴趣：** 发现与自己关心问题相关的智者
- **精通：** 学会使用搜索和书单工具精准定位
- **自主：** 可以选择被动接受推荐或主动搜索

#### 可供性：A4-搜索智者和著作

**级别：** 主要
**启用的操作：** 通过关键词搜索平台内容

**环境属性：**

| 属性 | 值 | 可被感知者 |
|----------|-------|----------------|
| 搜索框 | 顶部固定，带占位文本"搜索智者或著作…" | 人类 |

**感知-行动耦合：**

- **AI — 检测：** `<form role="search">` 带 `<input type="search">`
- **AI — 执行：** 通过 `/api/search?q={query}` 端点访问
- **AI — 验证：** 返回包含匹配智者和著作的 JSON 数据

**约束条件：**
- 搜索索引覆盖智者和著作数据，确保知识准确性（来自 real.md）

### MAS-3：深度参与与创作

> **故事主题：** 用户从被动接收内容，转向主动参与——记录思考、分享见解、构建个人知识体系。

**核心可供性序列：**

1. **撰写书评/笔记** — 对著作撰写个人解读（感知："写笔记"按钮）
2. **参与讨论** — 阅读他人书评并互动（感知：评论区）
3. **AI 对话助手** — 与 AI 讨论智者思想和著作（感知：对话入口）
4. **构建阅读轨迹** — 查看个人阅读历史与收藏（感知：个人中心）

**意义闭合：**
用户从"读者"成长为"创作者"——在人类文明长河中留下自己的作品。

**内在动机：**
- **兴趣：** 表达自己的理解并与他人交流
- **精通：** 写作质量随练习提升
- **自主：** 完全控制创作内容和分享范围

## 3. 环境约束

### 3.1 物理约束

| 约束 | 对可供性的影响 |
|------------|----------------------|
| 屏幕尺寸（移动端/桌面端） | 移动端分类导航改为底部 Tab 或汉堡菜单，智者卡片改为单列 |
| 输入方法（触屏/鼠标/键盘） | 搜索和笔记编辑支持多种输入方式 |
| 网络延迟 | 智者卡片和著作列表支持骨架屏加载，AI 对话支持流式输出 |

### 3.2 结构约束

| 约束 | 对可供性的影响 |
|------------|----------------------|
| 数据模型（智者、著作、用户） | 定义了分类浏览、搜索和推荐的数据基础 |
| 访问控制 | 收藏、写笔记、AI 对话需要登录；浏览和搜索无需登录 |
| 状态依赖 | 用户必须先有收藏行为，才能构建个人阅读轨迹 |

### 3.3 安全约束（来自 real.md）

| 约束 | 可供性限制 |
|------------|----------------------|
| 版权合规 | 智者和著作数据须标注来源《聪明的阅读者》，引用遵守规范 |
| 隐私保护 | 用户收藏和笔记数据加密存储，非本人不可见 |
| 内容审核 | 书评/笔记发布前须经管理员审核，防止不当信息 |
| 知识准确性 | 智者和著作信息须经人工确认，确保推荐可靠性 |

## 4. 感知通道

### 4.1 视觉可供性（人类）

| 元素 | 感知到的操作 | 设计模式 |
|---------|-----------------|----------------|
| 分类 Tab 栏 | 切换智者分类浏览 | 水平标签，选中态下划线高亮 |
| 智者卡片 | 点击查看详情 | 卡片圆角阴影，hover 上浮效果 |
| 搜索框 | 输入搜索内容 | 顶部固定带搜索图标，焦点态展开 |
| 收藏按钮 | 点击收藏 | 心形图标，空心↔实心切换动画 |
| "写笔记"按钮 | 打开编辑器创作 | 醒目按钮，带笔形图标 |

### 4.2 语义可供性（AI）

| 元素 | 语义标记 | 感知到的操作 |
|---------|----------------|------------------|
| 智者卡片 | `<article data-wise-person-slug="...">` | 访问智者详情 |
| 分类导航 | `<nav role="tablist" aria-label="智者分类">` | 按分类筛选 |
| 搜索 | `<form role="search">` | 执行搜索查询 |
| 收藏 | `<button data-action="bookmark" data-id="...">` | 添加/移除收藏 |
| 分页 | `<nav aria-label="分页导航">` | 翻页浏览 |

### 4.3 跨模态可供性

| 可供性 | 人类通道 | AI 通道 | 统一操作 |
|------------|---------------|------------|----------------|
| 搜索智者 | 输入关键词+回车 | GET `/api/search?q=` | 内容检索 |
| 分类浏览 | 点击分类 Tab | GET `/api/wise-persons?category=` | 数据筛选 |
| 收藏内容 | 点击收藏按钮 | POST `/api/bookmarks` | 数据持久化 |
| 获取推荐 | 点击 AI 助手 | POST `/api/ai/recommend` | 智能推荐 |
| 提交书评 | 填写编辑器+点击发布 | POST `/api/reviews` | 内容创建 |

## 5. 反馈机制

### 即时反馈（< 100ms）

| 操作 | 人类反馈 | AI 反馈 |
|--------|---------------|-------------|
| 点击分类 Tab | Tab 高亮切换，卡片淡入 | URL 参数更新事件 |
| 点击收藏 | 心形图标填色动画 | 按钮状态变化事件 |
| 搜索输入 | 实时搜索建议下拉 | 输入值变化事件 |
| 卡片 hover | 卡片轻微上浮+阴影加深 | CSS 类切换事件 |

### 进度反馈（< 1s）

| 操作 | 人类反馈 | AI 反馈 |
|--------|---------------|-------------|
| 搜索提交 | 搜索结果骨架屏 | 待处理状态 |
| 页面切换 | 路由过渡动画 | 页面加载事件 |
| AI 对话 | 消息"正在输入…"指示器 | WebSocket 连接等待 |
| 书评提交 | "发布中…"按钮禁用状态 | 表单提交进行中 |

### 完成反馈（< 5s）

| 操作 | 人类反馈 | AI 反馈 |
|--------|---------------|-------------|
| 搜索完成 | 搜索结果列表展示 | 搜索结果数据返回 |
| 收藏成功 | 短暂 toast "已收藏" | 收藏状态确认响应 |
| AI 回复完成 | 完整回复展示 | 流式数据闭合事件 |
| 书评发布 | "发布成功"提示+跳转到笔记区 | 书评 ID 和状态返回 |

## 7. 非可供性（明确禁止的动作）

| 禁止的动作 | 原因 |
|----------------|----------------------|
| 用户直接编辑智者和著作信息 | 内容来自《聪明的阅读者》，须经管理员审核修改（版权合规） |
| 未登录状态下收藏/写笔记 | 需要用户身份关联数据（隐私保护） |
| 书评中发布不当信息 | 须经内容审核机制过滤（UGC 审核） |
| AI 助手给出未经证实的知识断言 | AI 回答须标注"仅供参考"，引用平台内数据源（知识准确性） |
| 批量导出智者和著作数据 | 防止数据爬取和版权侵犯（版权合规） |

## 9. 附录

### 9.1 可供性理论参考

本文档采用 Gibson 可供性理论框架，核心理念：

- **可供性**是环境属性与智能体行动能力之间的关系，独立于智能体的感知而存在
- **MAS（最小可供故事）** 替代传统 MVP：关注"故事完整性"而非"功能列表"
- **感知-行动耦合**：每个可供性都定义了人类和 AI 双通道的感知与行动方式

### 9.2 术语表

| 术语 | 定义 |
|------|------|
| 可供性（Affordance） | 环境提供的行动可能性 |
| MAS（最小可供故事） | 最小但有意义的完整行动叙事 |
| 能指（Signifier） | 指示可供性存在的感知线索 |
| 感知-行动耦合 | 智能体感知环境并执行行动的过程 |
| 智能体（Agent） | 在环境中行动的主体（人类或 AI） |

### 9.3 传统功能需求映射（供参考）

| MAS 故事 | 传统功能列表 | 优先级 |
|----------|----------------|--------|
| MAS-1 探索智者世界 | 智者分类浏览、智者详情页、问题导览页 | P0 |
| MAS-2 发现阅读路径 | 全局搜索、书单推荐系统、收藏功能 | P1 |
| MAS-3 深度参与创作 | Markdown 编辑器、书评系统、AI 对话助手、个人中心 | P2 |
