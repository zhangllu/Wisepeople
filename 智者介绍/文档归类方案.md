## 部署到网站的方案

### 数据结构设计

在 `src/types/index.ts` 中添加新的接口：

```typescript
// 智者详细资料
export interface WisePersonDetail {
  slug: string
  name: string
  nameEn?: string
  // 01-智者介绍内容
  introduction: {
    whoIsHe: string              // 他是谁
    coreThoughts: {              // 核心思想
      title: string
      description: string
    }[]
    essence: string              // 他的底色
    relevance: string            // 当代意义
    works: {                     // 主要著作
      title: string
      year?: number
      summary: string
      difficulty: 1 | 2 | 3 | 4 | 5
    }[]
    questions: string[]          // 对话问题
    comparisons?: {              // 与其他智者对比
      name: string
      relationship: string
    }[]
  }
  // 02-原始资料元数据（不直接展示，仅供核对）
  metadata?: {
    sources: string[]            // 来源链接
    lastUpdated: string          // 更新日期
    contributor?: string         // 贡献者
  }
  // 03-认知方式（用于AI扮演）
  cognitiveStyle?: {
    primary: CognitiveMode       // 主要认知方式
    secondary?: CognitiveMode    // 次要认知方式
    thoughtProcess: string       // 思维路径
    questioningPath: string[]    // 提问路径
    oppositions: string[]        // 明确反对的东西
  }
}

export type CognitiveMode =
  | "thought-experiment"     // 思想实验
  | "symbolic-thinking"      // 符号思考
  | "experimental-science"   // 实验科学
  | "computational-simulation" // 计算模拟
  | "fieldwork"             // 田野调查
  | "humorous-narrative"     // 幽默叙事
  | "story-narrative"        // 故事叙事
  | "literary-aesthetics"    // 文采美感
  | "visual-aesthetics"      // 视觉美感
```

### 文件存放位置

#### 选项1：作为静态资源（推荐用于初期）

```
project-template-student-deploy_zl/
├── public/
│   └── wise-persons/
│       ├── plato/
│       │   ├── index.md          # 智者介绍
│       │   ├── sources.md        # 原始资料
│       │   └── cognitive.md       # 认知方式
│       ├── kant/
│       └── ...
```

**优点**：
- 不需要修改构建配置
- 可以直接通过URL访问
- 易于更新和维护

**缺点**：
- 需要前端解析Markdown

---

#### 选项2：作为TypeScript数据（推荐用于生产）

```
project-template-student-deploy_zl/
├── src/
│   ├── data/
│   │   ├── wise-persons/
│   │   │   ├── plato.ts
│   │   │   ├── kant.ts
│   │   │   └── index.ts          # 统一导出
│   │   └── wise-person-details.ts # 详细资料
```

**优点**：
- 类型安全
- 构建时优化
- 可以直接导入使用

**缺点**：
- 需要将Markdown转换为TypeScript对象
- 更新时需要重新构建

---

#### 选项3：混合方案（推荐）

```
project-template-student-deploy_zl/
├── src/
│   ├── data/
│   │   ├── wise-persons.ts        # 基础数据（现有的mock-data.ts）
│   │   └── wise-person-details/   # 详细Markdown文档
│   │       ├── plato.md
│   │       ├── kant.md
│   │       └── ...
```

**优点**：
- 基础数据保持TypeScript（类型安全）
- 详细内容用Markdown（易于编辑）
- 灵活性高
