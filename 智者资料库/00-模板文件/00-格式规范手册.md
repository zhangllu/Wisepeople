# 智者资料库 - 格式规范手册

> 本手册定义所有文档必须遵循的格式规范
> 最后更新：2026年5月1日
> 版本：v1.0

---

## 总体原则

### 1. 一致性原则
所有同类文档必须使用相同的格式和结构

### 2. 可读性原则
文档要易于阅读，易于查找信息

### 3. 可维护性原则
文档要易于更新和扩展

### 4. 可验证性原则
所有信息必须有明确来源

## 文件命名规范

### 目录命名
```
智者资料库/
├── [slug]/              # 智者目录名
├── 00-模板文件/         # 模板文件
└── *.md                # 说明文档
```

### slug命名规则

#### 西方智者
- 格式：全名小写，连字符分隔
- 示例：
  - `plato`
  - `immanuel-kant`
  - `jean-jacques-rousseau`
  - `paul-graham`
  - `james-march`

#### 东方智者
- 格式：拼音小写，连字符分隔
- 示例：
  - `laozi`
  - `zhuangzi`
  - `cao-xueqin`

### 文档命名

```markdown
01-智者介绍.md
02-原始资料.md
03-认知方式.md
04-人格卡.md（可选）
```

## Markdown格式规范

### 标题层级

```markdown
# 一级标题（智者中文名）
## 二级标题（主要章节）
### 三级标题（子章节）
#### 四级标题（细分，尽量避免）
```

**规则**：
- 一级标题只使用一次（文档开头）
- 二级标题用于主要章节
- 三级标题用于子内容
- 避免使用四级及以下标题

### 强调格式

```markdown
**加粗**：用于关键词、重要概念
*斜体*：用于外文术语、书名
`代码`：用于强调、文件名、命令
~~删除线~~：一般不使用
```

**规则**：
- 书名使用 `《书名》` 格式（中文）或 `"Book Name"`（英文）
- 外文术语首次出现时用 `*斜体*`，后续直接用中文
- 重要概念首次出现时用 **加粗**

### 列表格式

```markdown
- 无序列表使用 `- `
  - 嵌套缩进2个空格
  - 再嵌套再缩进2个空格

1. 有序列表使用 `1. `
   1. 嵌套使用不同标记
   2. 或缩进2个空格
```

**规则**：
- 列表项之间不留空行（除非需要分段）
- 嵌套列表缩进2个空格
- 有序列表嵌套时使用不同数字标记

### 引用格式

#### 行内引用
```markdown
> "这是一句引用"
> —— 作者，《著作名》
```

#### 脚注引用
```markdown
这是一句需要引用的话¹。

---

脚注标记：
¹《著作名》— 相关内容 [📖 原文]
```

**规则**：
- 使用上标数字 ¹ ² ³
- 脚注统一放在章节末尾
- 标注来源：[📖 原文] 表示有原文链接

### 代码块

```markdown
```
代码内容
```
```

**规则**：
- 需要时指定语言：```python
- 示例代码使用代码块
- 命令行指令使用代码块

### 表格格式

```markdown
| 列1 | 列2 | 列3 |
|-----|-----|-----|
| 内容1 | 内容2 | 内容3 |
```

**规则**：
- 列名使用短名称
- 对齐方式默认左对齐
- 表格前后留空行
- 复杂表格考虑是否需要简化

### 分隔线

```markdown
---
```

**规则**：
- 在主要章节之间使用
- 前后留空行
- 不滥用（通常用于区分不同主题）

## 特殊格式规范

### 日期格式

```markdown
## 智者姓名｜生卒年

示例：
## 柏拉图｜约公元前427年—前347年
## 康德｜1724—1804
```

**规则**：
- 年份使用全称（不用'24）
- 精确到年的用年，不精确的用世纪
- 西方智者标注中西历
- 东方智者标注朝代和年份

### 著作信息格式

#### 在智者介绍中
```markdown
| 书名 | 出版年 | 一句话 | 难度 | 入门推荐 |
|------|--------|--------|------|----------|
| 《书名》 | 年份 | 核心内容 | ★★☆ | ⭐ 首选 |
```

#### 在原始资料中
```markdown
### [分类]
- 《著作名》（[Orig. Name]）：[简短描述]
- 《著作名》（[Orig. Name]）：[简短描述]
```

### 难度标注

```markdown
★☆☆：入门级，任何人可读
★★☆：需要一定背景知识
★★★：需要专业背景或多次阅读
★★★★：专业性强，难度较大
★★★★★：非常困难，需要深厚基础
```

### 链接格式

```markdown
- 文字链接：[链接文字](URL)
- 引用链接：[^1]
- 脚注：¹

在脚注区：
[^1]: [说明文字](URL)
```

**规则**：
- 优先使用HTTPS链接
- 链接要有描述性文字
- 重要链接同时在脚注中标注

## 内容编写规范

### 语言风格

#### 中文表达
- 使用简体中文
- 避免使用港台用语（除非引用）
- 专业术语首次出现时注原文
- 避免过度使用英文缩写

#### 英文表达
- 专有名词首字母大写
- 著作名使用斜体或书名号
- 人名使用全称（首次出现时）

### 引用规范

#### 直接引用
```markdown
> "原文内容"
> —— 作者，《著作名》
```

#### 间接引用
```markdown
作者在《著作名》中提出……
```

#### 转述
```markdown
根据作者的观点……
```

### 原则
- 直接引用必须准确
- 间接引用要注明来源
- 避免过度引用
- 重要引用要标注出处

## 图片和媒体规范

### 图片命名

```
[slug]-[类型]-[序号].png

示例：
plato-portrait-01.png
laozi-diagram-01.png
```

### 图片引用

```markdown
![图片描述](path/to/image.png)

带尺寸：
![图片描述](path/to/image.png =300x200)
```

**规则**：
- 图片要有描述性alt文本
- 优先使用PNG格式
- 图片文件要放在合适目录
- 图片大小要合理（< 500KB）

## 常见问题

### Q1: 如何处理不确定的信息？
A: 使用"约""疑似""据传"等词语标注，并在原始资料中说明争议。

### Q2: 如何处理多来源的信息冲突？
A: 在原始资料中记录不同来源，在智者介绍中选择最可靠的说法。

### Q3: 如何处理智者有多个名字的情况？
A: 在文档开头说明："又称XXX，英文名XXX，本名XXX"。

### Q4: 如何处理生卒年不确定的情况？
A: 使用"约YYYY年"或"YYYY年前后"。

### Q5: 如何处理著作年代不确定的情况？
A: 使用"约YYYY年"或"YYYY年前后"，并说明原因。
